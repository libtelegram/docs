# Routes

In most scenarios, you will want to wait for a specific message or command from the user and respond to it. You can use Routes to process messages. To create a new Route call a route type on bot instance with trigger and handlers as params. The route types are:

  • command
  • text
  • photo
  • edited
  • inline
  • callback
  • poll
  • shiping
  • sticker
  • location

For example:

bot.command('start', (usr) => usr.reply('bot commands: \
  - /hello - get greetings'));
bot.text('hello', (usr) => usr.reply('üëã Hello!'));

# Triggers

Triggers is the optional first parameter for a route. Trigger types can be string, RegExp, array of string or array of RegExp. If you are expecting exactly the same message from the user, then use a string or an array of strings trigger. LibTelegram will go to the array and check if one of the strings matches the message.
